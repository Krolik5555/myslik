# Шаблон запроса фичи / правки

Скопируй блок ниже и заполни. Чем точнее — тем меньше раундов «не так».
Пустые поля пропускай. Для визуала **скрин с пометками > слов**.

```
## Что хочу (одно предложение)
…

## Где (вкладка / часть)
вкладка: Сегодня | Inbox | Задачи | Заметки | Календарь | Доска | Корзина
часть:  узел | хаб-область | связь | поповер узла | паутина(граф) |
        список-дерево | быстрый захват | палитра Ctrl+K | титлбар | сайдбар | таймер

## Поведение по шагам
1. когда я …
2. должно …
3. (граничный случай) если пусто/много/ошибка — …

## Критерий готовности (Done when)
- [ ] …
- [ ] …

## Визуал-референс
- скрин: (приложи / опиши «как сейчас» → «как надо»)
- или: «дай 3–4 варианта, выберу» (для нового вида, где сам не уверен)

## Не трогать / ограничения
- сохранить: …
- производительность/анимация: …
```

---

## Памятка по точным формулировкам

- **Скрин с пометками** снимает 90% догадок для визуала. Обведи/нарисуй стрелку.
- **Критерий готовности** = и ТЗ, и тест. «Готово, когда клик по области в
  Сегодня открывает её задачи; пустой список не показывает каретку».
- **Галерея вариантов** — когда не уверен «как должно выглядеть»: проси несколько
  вариантов и выбирай (как с фоном паутины), а не описывай словами несуществующее.
- **Глоссарий выше** — называй термин из списка, чтобы не было неоднозначности.
- **Крупное** — сначала короткий план (plan mode), правка плана, потом код.

## Дев-режим для быстрой проверки
- Превью в браузере (без перезапуска реального аппа): сервер `planner-ui` (:18901).
- `?dev` — всегда свежие демо-данные (граф/виды наполнены).
- `?view=tasks|notes|today|calendar|board|bin` — открыть сразу нужный вид
  (`?view=notes` — паутина заметок). Работает и в реальном аппе.
- `?light` — светлая тема (только в dev).
- Пример: `index.html?dev&view=notes&light`
